# Contributing to African Commodity Markets

This project is focused on solving one problem: helping Kenyan coffee farmers get fair prices through price protection services. Contributions should align with this focus.

## Getting Started

1. **Fork and clone the repository**
2. **Make one focused change** - solve a real problem, not add features
3. **Create a simple pull request**
4. **Wait for manual review**

## Project Philosophy

We follow a "simple as possible" approach:

- **One problem, one solution** - We focus on Kenyan coffee price protection
- **Manual process with software** - Founder reviews every contribution
- **Remove complexity** - Start simple, stay simple
- **Do things that don't scale** - Manual processes with software support

## What to Contribute

### Good Issues
- Fix bugs in the existing price display
- Improve UI clarity
- Add price protection consultation workflow
- Optimize existing functionality
- Improve error handling
- Enhance existing API documentation

### What Not to Contribute
- New commodities (we focus on Kenyan coffee only)
- Complex dashboards or charts
- Automated scaling features
- Platform features
- Multiple user types

## Development Workflow

1. **Check the current status** before starting
2. **Talk to the founder** about your planned changes
3. **Make one focused change** at a time
4. **Keep documentation minimal and clear**
5. **Submit a simple pull request**

## Code Standards

- **Keep it simple** - Remove unnecessary complexity
- **Use existing patterns** - Mimic the existing code style
- **Add no comments** unless explicitly requested
- **Focus on solving the problem**
- **One change per PR**

## Testing

Run the build to verify your changes:
```bash
pnpm build
```

## Questions?

If you have questions, create an issue and the founder will manually review and respond.

This is a service, not a platform. We work slowly and deliberately.
